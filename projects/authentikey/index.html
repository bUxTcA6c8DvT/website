Authentikey

Your tools, your keys, for life.

The Product

Authentikey is a security code generating app for users big into security. Security is a big topic today as 2 factor auth is now mandated by many apps such as Chrome Web Store and the Play Store. But to get security at its best one needs to enable two factor auth on many different accounts. This creates a conundrum as managing all of your accounts can be difficult in itself. Without a good app to keep track of them all you could very well lock yourself out of your own account! This project’s goal was to design a solution for a super user’s case of working with lots of two factor auth accounts. This is a mobile app geared for both your average user and super users of two factor auth.

Competitive Audit

Looking at several apps: Google Authenticator, Microsoft authenticator, and Authy.

There’s a lot of room in the market for a good authenticator app. While there are several big brands out there that support Security Code generating apps, none of them seem to expect a user to have that many accounts. While this may be true currently, it is certainly not for super users and for the increasing trend of two factor requirements. Currently 57% of companies around the world offer two factor services according to Last Pass. And large companies such as Amazon, Apple, Dropbox, Facebook, Google, Instagram, and Intuit already offer such services. There’s 7 accounts already. What a trend to be missing out on.

Here’s some market opportunities for a good authenticator app:

  • Lack of search and filters for when you have many accounts.
  • Many of the existing options lack auth support for non-standard schemes such as steam or authy
  • Lack of customization options. It can be hard to find the accounts because you can’t customize their display. So if you have a lot they all look the same.

These are the target user groups that are defined.

Super User:

  • A user with many accounts 20+
  • A user who cares a lot about security
  • A user who needs the app every time they login to a service. Want it to be quick and functional.
  • Want to know how much time they have left to use a security code
  • Millenial and living in urban and suburban areas.

Some of these are the same as the average user use case:

Average User:

  • A user with some accounts 5+
  • A user who cares a lot about security
  • A user who needs the app every time they login to a service. Want it to be quick and functional.
  • Want an easy to navigate interface and quick onboarding process.
  • Ages 16-85.
  • Living in urban and suburban areas.

Key Features

Both of these user types hold security of high importance, but ease of use is key if we want to aim for the average use case in addition to drawing in more users.

Some key features identified for these user groups are:

Features:

  • Provide search and filter options
    Search and filters are front and center. Filter by categories and search for usernames or entities. Search and filtering is key to solving the user problem of having lots of accounts.
  • Provide customization of visual elements
    Visual elements help users find the account they’re looking for, fast. Customize a photo of the entity you’re saving a two factor auth code for.
  • Provide a visible signifier for the time left on a security code’s use
    Once you’ve selected an account see how much time you have left to use the code with a handy time bar detailing how long until the code expires.

Conclusion

Most users wanted to be able to find what they’re looking for quickly. This makes sense as you don’t want two factor auth to be a burden but rather be a protective step. Some next steps may be to determine if there’s a way to provide pictures for each service ahead of time instead of requiring users to upload custom photos. (While still keeping the option there.) It would also be beneficial to test this mockup for the extreme use case of 50+ accounts to ensure that the categories for the filters are correctly set, and make it easier to manage accounts. There’s also the complication of having more than one account for a single service. In this case the custom image option allows the user to upload a separate image for each account. Although in the majority of the cases this picture could potentially be the same. Thus it would be beneficial to design a way to differentiate accounts with the same image displayed from the home screen.
